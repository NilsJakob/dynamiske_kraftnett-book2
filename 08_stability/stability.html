Stabilitet#

For å forstå stabilitet i dynamiske kraftnett må vi definere tidsaspektet. Hittil har vi jobbet med tiddserier og tidsserieanalyse, for å modellere kraftnettet. Vi skal nå bevege oss inn på transient-, spenning- og frekvensstabilitet.

Vi starter med den generiske sinusoide funksjonen og husker Eulers formel: \(e^{j\theta}=cos\theta+jsin\theta\).

I denne boken bruker vi MKS-enheter og IEEE-standarder så langt det lar seg gjøre Meter–Kilogram–Sekund (MKS) systemet er et enhetssystem der de grunnleggende måleenhetene er:

Lengde: meter (m)

Masse: kilogram (kg)

Tid: sekund (s)

MKS betyr at vi bygger alle fysiske størrelser fra meter, kilogram og sekund – og det er grunnlaget for SI-enhetene vi bruker i kraftsystemer.

\[ 1 V = \frac{kg\cdot m^2}{s^3\cdot A} \]

Alle fysiske størrelser i kraftsystemer er egentlig i SI-enheter (MKS-basert):

Spenning: Volt Strøm: Ampere Effekt: Watt Impedans: Ohm

Per enhet normaliserer MKS-verdier ved å dividere med en valgt baseverdi, for å skape dimensjonsløse tall uten enheter

\[ x_{pu} = \frac{x_{MKS}}{x_{base}} \]

Baser#

Fra basene \(S_{base}(VA)\) og \(V_{base}(Volt)\) kan vi utlede:

\[I_{base} = \frac{S_{base}}{\sqrt{3}V_{base}}\]
\[Z_{base} = \frac{V^2_{base}}{S_{base}}\]

Tidsdomene og fasor#

Tidsdomenerepresentasjon for spenning kan beskrives som: \(v(t)=\hat{V} \cdot cos(\omega t)\)

Små bokstaver (v) brukes til å representere instansverdier som varierer som funksjon av tiden (t).

Mens stor bokstav (V) er konstant.

Representasjon av tiden og konstanter er ikke standardisert som ved (MKS, SI og IEEE), men et produkt av konvensjoner; hva man over tid har blitt enige om å uttrykke noe på. Dette er også en god beskrivelse av forskjellen på standarder og konvensjoner.

Der \(\hat{V}\) er spenningens amplitude, frekvensen (f) er 50 Hz (50 svingninger i sekundet).

I python-scriptet under er V_ampl den konstante spenningen mens v forandrer seg med tid (t)

import numpy as np
import matplotlib.pyplot as plt

# Parameters
V_ampl = 325            # Peak voltage [V]
f = 50                 # Frequency [Hz]
omega = 2*np.pi*f      # Angular frequency
theta_v = np.pi/6     # Phase angle (30 degrees)

# Time vector
t = np.linspace(0, 0.1, 1000)

# Voltage signal
v = V_ampl * np.cos(omega * t + theta_v)

# Plot
plt.figure(figsize=(8, 3))
plt.plot(t, v)
plt.xlabel("Time (s)")
plt.ylabel("Voltage v(t) [V]")
plt.title(r"$v(t) = \hat{V} cos(ωt)$")
plt.grid()
plt.show()

I kraftsystemer beskriver man ofte cos(\(\omega t\)), fordi dette viser projeksjonen av spenningen v, i cosinusplanet og gir verdien 1 der v(t) = 0. Da er v(0) lik V den konstante, som er det samme som amplitudeverdien \(\hat{V}\). På denne måten beskriver cosinus systemet ved kontinuerlig drift som et steady-state perspektiv. Steady-state perspektivet er også en viktig faktor når vektoren (V) roterer i tidsdomenet og blir en fasorrepresentasjon.

Tidsdomene (bølgeform) og roterende vektor (fasor)#

Fasorrepresentasjonen blir særlig aktuell ved faseforskyvning, nyttig for å vise faseforskyvning av spenning (som i trefasesystemer) eller strøm (på grunn av passive komponenter:

\[v(t)=\hat{V} \cdot sin(\omega t) \Longleftrightarrow \bar{V} = \hat{V} \angle \theta\]

Vi bruker benevnelsen \(\bar{V}\) for å vise at det er et kompleks tall med en utstrekning (amplitude - \(\hat{V}\)) og fase (\(\theta\)). Utstrekningen er også magnituden eller absoluttverdien |V|, der fasen eller vinkelen er argumentet til det komplekse tallet.

I kraftsystemer er konvensjonen å jobbe med cosinusrepresentasjon på grunn av steady-state perspektivet nevnt ovenfor.

\[ \Re\{e^{j\omega t}\} = \cos(\omega t) \]

  • ingen ekstra fase

  • fasoren peker langs x-aksen ved \(t = 0\)

\[ \Im\{e^{j\omega t}\} = \sin(\omega t) \]

Men da skjer dette:

  • ved \(t = 0\): verdi = 0

  • du “ser” ikke vektoren direkte

Når du bruker realdelen:

\[ v(t) = \Re\{V e^{j\omega t}\} \]

blir koblingen:

\[ V = |V| e^{j\theta} \]

helt direkte.

Resultat:#

\[ v(t) = |V| \cos(\omega t + \theta) \]

  • fasen \(\theta\) er akkurat den samme som i fasoren

  • ingen ekstra forskyvning

Med sinus må du skrive:

\[ v(t) = |V| \sin(\omega t + \theta) \]

som egentlig er:

\[ = |V| \cos(\omega t + \theta - 90^\circ) \]

altså:

\[ V = |V| e^{j(\theta - 90^\circ)} \]

Root Mean Square (RMS)#

“RMS står for ‘root mean square’, og er den effektive verdien av et signal. For en vekselspenning som går opp og ned, kan vi ikke bruke vanlig gjennomsnitt, fordi det blir null. I stedet kvadrerer vi signalet, tar gjennomsnittet, og tar kvadratroten tilbake — og da får vi en verdi som tilsvarer den DC‑spenningen som gir samme effekt. For en sinus gjelder:

\[V_{RMS} = \frac{\hat{V}}{\sqrt{2}}\]

Så når vi sier 230 volt i stikkontakten, er det RMS — altså den verdien som faktisk bestemmer hvor mye energi som leveres.

Men den faktiske sinusen er \(v(t) = 325sin(\omega t)\)

For et signal x(t): $\(X_{RMS} = \sqrt{\frac{1}{T}\int^T_0x^2(t)dt}\)$

Derfor får vi $\(V_{RMS} = \frac{\hat{V}}{\sqrt{2}}\)\( og \)\(I_{RMS} = \frac{\hat{I}}{\sqrt{2}}\)$

Det vi kan trekk av spenning er 230 V, men amplitudesignalet er \(\hat{V} = V \cdot \sqrt{2} = 230 \cdot \sqrt{2}\approx 325\) V

\(cos \theta\) blir brukt til å foklare faseforskyvning og kjenner vi den kan vi gjøre effektberegninger direkte: $\(P = V_{RMS}I_{RMS}cos\theta\)$

import numpy as np
import matplotlib.pyplot as plt

# Data
t = np.linspace(0, 2*np.pi, 500)
v = np.sin(t)

# RMS value
v_rms = np.sqrt(np.mean(v**2))

# Plot
plt.figure()
plt.plot(t, v, label='Signal (sinus)')
plt.axhline(v_rms, linestyle='--', label='RMS-verdi')
plt.axhline(-v_rms, linestyle='--')

plt.title('Visualisering av RMS')
plt.xlabel('Tid')
plt.ylabel('Amplitude')
plt.legend()

\(\theta_v\) i trefase?#

\[ \theta_v = \omega t + \phi \]

  • \(\omega t\) → tidsutvikling

  • \(\phi\) → fasevinkel

  • \(\theta_v\) → instantan vinkel for spenningen

Dette brukes til å beskrive signalene i tidsdomene

Trefasespenninger med \(\theta_v\)#

Standard balansert system:

\[\begin{split} \begin{aligned} v_r(t) &= V_m \cos(\theta_v) \\ v_s(t) &= V_m \cos(\theta_v - 120^\circ) \\ v_t(t) &= V_m \cos(\theta_v - 240^\circ) \end{aligned} \end{split}\]

kompaktform:

\[ v_k(t) = V_m \cos(\theta_v - k \cdot 120^\circ), \quad k = 0, 1, 2 \]

Alt styres av én vinkel \(\theta_v\)

  • Alle tre fasene “roterer sammen”

  • Forskyvningen er konstant (\(120^\circ\))

Kobling til fasor (roterende vektor)#

I fasorform skriver vi:

\[\begin{split} \begin{aligned} V_r &= V \angle 0^\circ \\ V_s &= V \angle -120^\circ \\ V_t &= V \angle -240^\circ \end{aligned} \end{split}\]

Tolkning:

  • Fasorene er konstante

  • Rotasjonen ligger i:

\[ e^{j\omega t} \]

Kobling mellom de to#

Tidssignal ↔ fasor:

\[ v_r(t) = \Re \{ V_r e^{j\omega t} \} \]

og

\[ \theta_v = \omega t \]

“Vi beskriver trefasesystemet med én felles vinkel \(\theta_v\).
Alle fasene er bare forskjøvet versjoner av samme signal, med \(120^\circ\) mellomrom.
Det betyr at hele systemet kan sees som én roterende størrelse.”

Kompakt skrivemåte#

\[\begin{split} v_{rst}(t) = V_m \begin{bmatrix} \cos(\theta_v) \\ \cos(\theta_v - 120^\circ) \\ \cos(\theta_v - 240^\circ) \end{bmatrix} \end{split}\]
import numpy as np
import matplotlib.pyplot as plt
import matplotlib.animation as animation

# Parameters
t = np.linspace(0, 2*np.pi, 200)
Vm = 1

# Figure with two panels
fig, (ax_time, ax_phasor) = plt.subplots(1, 2, figsize=(10, 4))

# --- Time domain ---
ax_time.set_xlim(0, 1)
ax_time.set_ylim(-1.5, 1.5)
ax_time.set_title("Tidsdomeneprojeksjon")
ax_time.set_xlabel("Tid")

line_a, = ax_time.plot([], [], label="va")
line_b, = ax_time.plot([], [], label="vb")
line_c, = ax_time.plot([], [], label="vc")
ax_time.legend()

# --- Phasor plot ---
ax_phasor.set_xlim(-1.2, 1.2)
ax_phasor.set_ylim(-1.2, 1.2)
ax_phasor.set_title("Roterende vektorer (fasor)")
ax_phasor.axhline(0)
ax_phasor.axvline(0)

vec_a = ax_phasor.arrow(0, 0, 0, 0)
vec_b = ax_phasor.arrow(0, 0, 0, 0)
vec_c = ax_phasor.arrow(0, 0, 0, 0)

# storage
ta, va_hist = [], []
tb, vb_hist = [], []
tc, vc_hist = [], []

def update(frame):
    global vec_a, vec_b, vec_c
    
    theta = frame
    
    # phases
    angles = [theta, theta - 2*np.pi/3, theta - 4*np.pi/3]
    
    va = Vm * np.cos(angles[0])
    vb = Vm * np.cos(angles[1])
    vc = Vm * np.cos(angles[2])
    
    # store history (normalized time)
    time_val = theta / (2*np.pi)
    ta.append(time_val); va_hist.append(va)
    tb.append(time_val); vb_hist.append(vb)
    tc.append(time_val); vc_hist.append(vc)
    
    # update time plot
    line_a.set_data(ta, va_hist)
    line_b.set_data(tb, vb_hist)
    line_c.set_data(tc, vc_hist)
    
    # remove old arrows
    vec_a.remove()
    vec_b.remove()
    vec_c.remove()
    
    # draw new phasors
    vec_a = ax_phasor.arrow(0, 0, np.cos(angles[0]), np.sin(angles[0]),
                           head_width=0.05)
    vec_b = ax_phasor.arrow(0, 0, np.cos(angles[1]), np.sin(angles[1]),
                           head_width=0.05)
    vec_c = ax_phasor.arrow(0, 0, np.cos(angles[2]), np.sin(angles[2]),
                           head_width=0.05)
    
    return line_a, line_b, line_c, vec_a, vec_b, vec_c

ani = animation.FuncAnimation(
    fig, update, frames=np.linspace(0, 2*np.pi, 100), interval=50
)

ani.save("trefase_fasor.gif", writer="pillow")
plt.close()

import numpy as np
import matplotlib.pyplot as plt
import matplotlib.animation as animation

# Parameters
t = np.linspace(0, 2*np.pi, 200)
Vm = 1

# Figure with two panels
fig, (ax_time, ax_phasor) = plt.subplots(1, 2, figsize=(10, 4))

# --- Time domain ---
ax_time.set_xlim(0, 1)
ax_time.set_ylim(-1.5, 1.5)
ax_time.set_title("Tidsdomeneprojeksjon")
ax_time.set_xlabel("Tid")

line_a, = ax_time.plot([], [], label="va")
line_b, = ax_time.plot([], [], label="vb")
line_c, = ax_time.plot([], [], label="vc")
ax_time.legend()

# --- Phasor plot ---
ax_phasor.set_xlim(-1.2, 1.2)
ax_phasor.set_ylim(-1.2, 1.2)
ax_phasor.set_title("Roterende vektorer (fasor)")
ax_phasor.axhline(0)
ax_phasor.axvline(0)

vec_a = ax_phasor.arrow(0, 0, 0, 0)
vec_b = ax_phasor.arrow(0, 0, 0, 0)
vec_c = ax_phasor.arrow(0, 0, 0, 0)

# storage
ta, va_hist = [], []
tb, vb_hist = [], []
tc, vc_hist = [], []

def update(frame):
    global vec_a, vec_b, vec_c
    
    theta = frame
    
    # phases
    angles = [theta, theta - 2*np.pi/3, theta - 4*np.pi/3]
    
    va = Vm * np.cos(angles[0])
    vb = Vm * np.cos(angles[1])
    vc = Vm * np.cos(angles[2])
    
    # store history (normalized time)
    time_val = theta / (2*np.pi)
    ta.append(time_val); va_hist.append(va)
    tb.append(time_val); vb_hist.append(vb)
    tc.append(time_val); vc_hist.append(vc)
    
    # update time plot
    line_a.set_data(ta, va_hist)
    line_b.set_data(tb, vb_hist)
    line_c.set_data(tc, vc_hist)
    
    # remove old arrows
    vec_a.remove()
    vec_b.remove()
    vec_c.remove()
    
    # draw new phasors
    vec_a = ax_phasor.arrow(0, 0,
                       np.cos(angles[0]), np.sin(angles[0]),
                       head_width=0.05, color='blue')

    vec_b = ax_phasor.arrow(0, 0,
                       np.cos(angles[1]), np.sin(angles[1]),
                       head_width=0.05, color='orange')

    vec_c = ax_phasor.arrow(0, 0,
                       np.cos(angles[2]), np.sin(angles[2]),
                       head_width=0.05, color='green')
    
    return line_a, line_b, line_c, vec_a, vec_b, vec_c

ani = animation.FuncAnimation(
    fig, update, frames=np.linspace(0, 2*np.pi, 100), interval=100
)

ani.save("trefase_fasor_farge.gif", writer="pillow")
